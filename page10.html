DX12 Circles

Current WIP

Project is currently in development. Pictures and description will be posted when complete

This project was developed as an exercise in DOD principles, Multithreading and Optimisation. Half of the circles are moving and collide with any stationary circles. The collision function is multithreaded by utilising a thread pool and data is accessed in a way that prevents cache misses on the CPU. Currently simulating 20K circles at ~30FPS with 90% percent of CPU time spent on collision checks. Rendered using a custom DirectX 12 engine in C++.
